## 五、项目怎么运作

1. **技能文件是核心**——所有策略知识都在 `.reasonix/skills/` 下。修改技能文件 = 让 Marsala 学会新东西。
2. **内部版和交付版分离**——内部版是完整版（含经验沉淀区、底层原理），交付版是精简版（只含对外服务内容）。
3. **同步规则**——改内部版后，必须同步到分发版。同步用 `Copy-Item`，排除 README。
4. **MEMORY.md** 是大脑——记录项目状态、信息来源、调用原则。
5. **ACKNOWLEDGMENTS.md** 是开源致谢——参考过的 GitHub/GitCode 仓库。
6. **`.reasonix/attachments/`** 是历史文章存档——不是项目的核心部分。

## 七、常见任务模式

1. **修改技能文件** → 改内部版 → 同步分发版（排除 README）
2. **更新版本号** → 改 MEMORY.md + README.md（内部版）→ 同步分发版 MEMORY.md（改路径）
3. **读奥美报告** → 用 pymupdf 提取 PDF 文本 → 提炼框架 → 写入对应技能文件
4. **搜索信息** → 优先用 360 搜索（`web_fetch`），小红书/B站用 `opencli`
5. **运行 Python 脚本** → 用 `python script.py`，不要用内联 `python -c`（引号转义问题严重）

## 八、经验教训（来自过往对话）

1. **写完必须读**——不能假定自己知道文件内容。用 `read_file` 确认。
2. **引号转义是常见错误**——PowerShell 下 `python -c` 的引号极易出错，改用 `.py` 文件。
3. **`old_string` 必须精确匹配**——`edit_file` 失败 90% 是因为空格、换行、省略号不匹配。
4. **他不喜欢废话**——直接做事，做完简洁汇报。
5. **他能看出你在敷衍**——如果你说"已写入"但实际没写，他会发现。
