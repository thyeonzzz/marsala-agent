# 对外输出规则

当 agent 使用者向**客户**展示能力或交付策略方案时，适用以下规则。

---

## 核心原则

同一套方法论，换一种语言。不是另写一份内容——是从内部版技能文件中提取方法论，剥掉内部操作语言，保留策略判断和论据。

---

## 剥离清单

对外输出中**不出现**以下内容：

- 六手架构称谓（"手一""手三""客户总监"等）
- 证据等级标签（E1-E6）
- 技能文件名称、内部版/交付版区分
- 校验规则编号（"校验一""校验三"等）
- agent 内部调度逻辑（"手三产出后传递给手六"）
- 对 agent 自身的操作指令（"你必须""禁止""注意事项"中以 agent 为对象的条目）
- 经验沉淀区内容
- 工具安装与调用细节（pymc-marketing、Agent Reach 等）

---

## 保留与转换

| 保留 | 转换方式 |
|------|---------|
| 方法论框架（矩阵、模型、分析维度） | 直接使用，不标注来源手 |
| 决策规则和判断标准（如"LTV:CAC ≥ 3:1"） | 保留规则，去掉推导过程 |
| 创意评估测试（去Logo测试、竞争复制测试等） | 作为分析方法呈现，不说"这是手五的校验" |
| 数据来源和局限性 | 文末统一列出，文中引数据时简要标注来源类型 |
| 不确定性和风险 | 保留，这是策略可信度的一部分 |

---

## 语气

- 用 **"我们"** 自称，不用"agent""Marsala"
- 直接给结论，再展开论据。不先说"我们做了以下分析"
- 不解释分析方法论（除非客户追问）
- 承认信息不足和边界——不编造

---

## 交付件参考结构

不强制套模板，但有效模式是：

1. **核心建议**（一页内说清楚）
2. **分析论证**（数据 → 洞察 → 判断，呈现分析结果而非分析过程）
3. **策略建议**（方向 + 理由 + 竞品对比 + 风险评估）
4. **创意方向**（策略到表达的转换、具体产出）
5. **风险与纠错**（关键不确定性、独特预测、纠错机制、试验成本承担）
6. **下一步**（需要品牌方确认和提供的事项）
7. **数据来源**（文末列出所有来源及局限性）

---

## 媒介策略的交付边界

媒介策划手的所有判断都是方向性、建议性的。交付件中的媒介内容严格限于以下范围：

**可以包含：**
- 平台选择方向与理由（为什么选小红书而非抖音）
- 平台角色分工（小红书做信任、抖音做声量、点评做收口）
- 内容策略方向（什么类型的内容在什么平台、面向什么人群）
- KOL 层级建议（头部做曝光、腰部做圈层、KOC 做搜索占位）——不指定具体达人
- 预算框架方向（内容制作 vs KOL vs 效果投放的大致比例）——标注为方向性建议
- 阶段节奏（先铺内容再放量还是同步进行）

**不包含：**
- 精确的预算分配金额或百分比
- 具体达人推荐名单
- 渠道 ROI 或转化成本估算
- CPM/CPC/CTR 等投放效果指标的预测值
- 广告投放操作建议（出价策略、定向设置、素材规格）

**强制标注：** 媒介相关内容必须标注"以下建议基于行业基准和公开案例推断，未基于品牌实际投放数据。具体执行需在投放测试后校准。不接受将方向性建议作为精确预算分配依据。"

> 背景：媒介策划是六手中数据依赖最重的一只手。在当前个人开发者环境下，渠道 ROI、投放效果等核心数据完全不可得。交付件中的媒介内容定位于"帮客户理清方向"，而非"替客户做投放决策"。实际的投放执行和优化，由客户或其代理商基于真实数据完成。
